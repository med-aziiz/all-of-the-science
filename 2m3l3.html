Lesson 3.3: Testing, Debugging & Version Control

This lesson covers essential practices to ensure your code works correctly, including testing, debugging, and using version control systems like Git.

1. Unit Testing

Definition: Testing small, individual parts of your code to verify correct functionality.

Example: Testing a function that calculates the area of a circle:

import math

def area_of_circle(radius):
    return math.pi * radius * radius

def test_area_of_circle():
    expected = math.pi * 3 * 3
    assert abs(area_of_circle(3) - expected) < 0.0001, "Test failed: Incorrect area calculation."

test_area_of_circle()
print("Unit test passed!")

2. Debugging

Definition: The process of identifying and fixing errors in your code.

Techniques:

  • Using print statements to display variable values.
  • Utilizing interactive debuggers (e.g., Python's pdb).
  • Explaining your code to yourself or a “rubber duck” to spot mistakes.
def divide(a, b):
    print("a =", a, "b =", b)
    return a / b

print("Result:", divide(10, 2))

3. Version Control

Definition: Tools and practices to track changes in your code over time, facilitating collaboration and rollback when needed.

Example using Git:

# Create a new repository
mkdir my_project
cd my_project
git init

# Create a file
echo "print('Hello, Version Control!')" > hello.py

# Add and commit the file
git add hello.py
git commit -m "Initial commit: add hello.py"

# Create and switch to a new branch
git branch feature-x
git checkout feature-x

# Modify the file and commit changes
echo "print('Feature X in progress')" >> hello.py
git add hello.py
git commit -m "Add feature X update in hello.py"
